Trabajo Realizado

Partes Implementadas

  • Estructura base HTML5 válida para todas las páginas.
  • Diseño Responsive Mobile First con Media Queries.
  • Página de Inicio (index.html) con listado estático.
  • Formularios completos en login.html y registrar.html con validaciones nativas.
  • Estética unificada mediante variables CSS y Flexbox.

Partes No Implementadas

  • Funcionalidad de Logout (Práctica 2).
  • Interacción dinámica (JavaScript) en botones y formularios (Práctica 2).
  • Persistencia de datos o subida real de imágenes.

Incompatibilidades y Problemas

Problema / Incompatibilidad Solución Aplicada
Confusión con efectos :hover en dispositivos táctiles (móviles). Se optó por utilizar estados visuales más claros y se comprendió que el hover es limitado en pantallas táctiles, priorizando el diseño visual estático en móvil.
Ajuste de imágenes de perfil circulares con diferentes proporciones. Uso de object-fit: cover para asegurar que la imagen siempre llene el círculo sin deformarse.
Sincronización de estilos compartidos entre Login y Registro. Además de las páginas de index y buscar que también comparten estilos comunes. Refactorización del CSS para extraer clases comunes (.form-container, .auth-form) y evitar duplicidad de código.
Reglas CSS ignoradas por errores tipográficos sutiles. Depuración de la hoja de estilos eliminando errores de sintaxis (dobles almohadillas en hexadecimales, espacios en unidades métricas como 40 px y nombres de clases incompletos).
Desconexión entre las etiquetas semánticas del DOM y los estilos CSS (ej. imágenes a tamaño real). Revisión de los archivos HTML para asegurar la correcta asignación de clases estructurales (como class="activity-detail-page") en las etiquetas <main> correspondientes.

Información Adicional

Se ha prestado especial atención a la accesibilidad mediante el uso de etiquetas semánticas y el contraste de colores para cumplir con los estándares de diseño multimedia.
